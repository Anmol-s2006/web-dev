CSS Detailed Notes (2025)

1. What is CSS?

Cascading Style Sheets (CSS) is a stylesheet language used to control the look and formatting of HTML documents. It lets you separate content from presentation, making your code cleaner and more maintainable.

2. How CSS Works

  • Browsers read HTML and apply CSS rules to elements, giving them color, spacing, layout, animation, etc.
  • CSS is cascading: If several rules match the same element, specificity and order determine which one applies.
Separation of Concerns: Keep structure (HTML) and presentation (CSS) separate for better reusability and readability.

3. CSS Syntax

selector {
  property: value;
  property2: value2;
}
  • Selector: Which elements the style will apply to (e.g. h1, .main)
  • Declaration: One or more property-value pairs (e.g. color: red;)

4. Ways to Use CSS

  • Inline CSS: <h1 style="color: green;">Text</h1>
  • Internal CSS: In the <style> tag inside <head> section
  • External CSS: Separate .css file linked by <link rel="stylesheet" href="file.css"> in <head>
Always use external CSS on real projects. This keeps style separate from structure and makes reusing styles across pages easy.

6. CSS Colors

  • Named Colors: red, blue, etc.
  • RGB: rgb(34,150,243)
  • RGBA: rgba(34,150,243, 0.5) (last value is opacity)
  • Hex: #2296f3
  • HSL: hsl(207, 85%, 54%)

7. Units

  • Absolute: px (pixels), cm, mm, in, etc.
  • Relative: em (relative to parent), rem (root em), %, vw (viewport width), vh (viewport height)

8. The CSS Box Model

  • Content: The content itself
  • Padding: Space between content and border
  • Border: Surrounds the padding (if any)
  • Margin: Space outside border
div {
  padding: 12px;
  border: 2px solid #1e2c3c;
  margin: 20px;
}

9. Display Property

  • block: Full width, new line (<div>, <p>)
  • inline: Flows in line (<span>, <a>)
  • inline-block: Like inline but can set width/height
  • none: Removes from display
  • flex: Flexible box container
  • grid: CSS Grid layout container

10. Positioning in CSS

  • static: Default, normal flow
  • relative: Offset from normal position
  • absolute: Positioned to the nearest ancestor with position (not static)
  • fixed: Stays fixed to screen
  • sticky: Scrolls and then sticks

12. Flexbox

Modern layout system for arranging items on a single axis — horizontal (row) or vertical (column).

.container {
  display: flex;
  flex-direction: row; /* or column */
  justify-content: center;
  align-items: center;
}
.item {
  flex: 1;
}
  • justify-content: Aligns items on main axis
  • align-items: Aligns items on cross axis
  • gap: Space between flex children

13. CSS Grid

Powerful 2D layout system for building advanced web layouts.

.container {
  display: grid;
  grid-template-columns: 1fr 1fr 2fr;
  grid-gap: 12px;
}

14. Responsive Design & Media Queries

@media (max-width: 800px) {
  body { font-size: 16px; }
  nav { display: none; }
}
  • Use relative units (em, rem, %, vw, vh).
  • Adapt layout with media queries for phones, tablets, desktops.

15. CSS Transitions & Animations

Transitions

div {
  transition: background 0.4s ease, color 0.2s linear;
}
div:hover {
  background: #baf3c2;
  color: #344b23;
}

Animations

@keyframes moveBox {
  0%   { left: 0; }
  100% { left: 200px; }
}
div {
  position: relative;
  animation: moveBox 2s infinite alternate;
}

16. Advanced Topics

  • Variables: :root { --main-color: #2ec4b6; }
  • calc(): width: calc(100% - 60px);
  • Selectors Level 4: :is(), :not(), etc.
  • Custom Fonts: @font-face
  • Transitions of transforms, filter, etc.
  • Browser prefixes: -webkit-, -moz-, etc.

17. Best Practices

  • Write modular, reusable CSS (use .class not #id for most styles)
  • Organize CSS with comments and logical groupings
  • Keep selectors as specific as needed but not more
  • Always validate syntax on validator.w3.org
  • Test in multiple browsers for compatibility
  • Use browser DevTools (F12) for debugging
  • Comment code: /* This is a comment */
